**# 互动大屏幕投票系统 (Interactive Display System)**

**## 项目说明**

本项目是为了企业内部培训时，需要进行投票互动而开发的一个web系统。

系统包含三个主要视图：

\*  ***\*公共页面\****：适配 PC 屏幕/大屏，系统启动后，在会议室大屏幕上展示签到二维码（未投票时），投票选项（投票进行时）、投票结果（投票结束后）和统计数据（整个活动结束后），该页面上没有按钮等操作按键，因为不会有人直接操作此页面。补充，投票进行时，右上角也会显示较小的二维码，支持用户中途加入签到。

\*  ***\*配置页面\****：适配 PC 屏幕，供管理员配置活动主题和投票内容（支持动态添加多次投票及投票内容），需密码访问。

\*  ***\*用户页面\****：适配手机屏幕，参会人通过扫描大屏页面的二维码签到后进入，进行答题、投票等互动操作，用户签到后，页面会显示“已签到”字样，并等待管理员开始投票，如果投票互动已经开始，用户再次签到，则直接进入投票页面。



**## 快速开始 (Quick Start)**



**### 环境要求**

\*  Python 3.9+



**### 安装与启动**

本项目提供了一个启动脚本 `run.py`，它会自动检测并安装所需的依赖包 (`requirements.txt`)。



在项目根目录下运行：



\```bash

python run.py

\```



如果需要手动安装依赖：

\```bash

pip install -r requirements.txt

python run.py

\```


**### 访问地址**

服务启动后（默认端口 8000），可以通过以下地址访问：



\*  ***\*大屏幕入口 (Display)\****: [http://localhost:8000/](http://localhost:8000/)

\*  ***\*管理员入口 (Admin)\****: [http://localhost:8000/admin](http://localhost:8000/admin)

\*  ***\*手机签到 (Mobile)\****: [http://localhost:8000/signin](http://localhost:8000/signin)



**## 功能特性**


**### 1. 签到系统**

\*  ***\*扫码签到\****：首页显示动态二维码，用户手机扫号码签到。

\*  ***\*实时统计\****：大屏幕实时显示已签到人数。

\*  ***\*身份识别\****：支持普通参会人和主持人身份（主持人需密码认证）。

\*  ***\*防重复\****：自动识别已签到用户，如果用户已经签到，则再次签到后，不会重复统计签到人数。如果已经开始投票互动，则再次签到后，直接进入投票页面。


**### 2. 后台管理**

\*  ***\*培训配置\****：配置活动主题和投票内容（支持动态添加多次投票及投票内容），需密码访问。

\*  ***\*互动控制\****：主持人并实时控制互动的开启、结束并显示结果、退出互动（大屏回到签到页面）。
