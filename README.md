**# 互动大屏幕投票系统 (Interactive Display System)**

**## 项目说明**

本项目是为了企业内部培训时，需要进行投票互动而开发的一个web系统。

系统包含四个主要视图：

\*  ***\*公共页面\****：适配 PC 屏幕/大屏，系统启动后，在会议室大屏幕上展示签到二维码（未投票时），投票选项（投票进行时）、投票结果（投票结束后）和统计数据（整个活动结束后），该页面上没有按钮等操作按键，因为不会有人直接操作此页面。补充，投票进行时，右上角也会显示较小的二维码，支持用户中途加入签到。

\*  ***\*配置页面\****：适配 PC 屏幕，供管理员配置活动主题和投票内容（支持动态添加多次投票及投票内容），需密码访问。

\*  ***\*用户页面-参会人\****：适配手机屏幕，参会人通过扫描大屏页面的二维码签到后，选择身份，填写姓名后进入，进行答题、投票等互动操作，用户签到后，页面会显示“已签到”字样，并等待主持人开始投票，如果投票互动已经开始，用户再次签到，则直接进入投票页面，同一次活动中，用户重复签到，签到人数不会重复统计。

\*  ***\*用户页面-主持人\****：适配手机屏幕，主持人通过扫描大屏页面的二维码签到后，选择身份，并输入密码后进入，主持人并实时控制活动的开始（显示签到页面）、选择开始某一个互动（投票）、结束并显示结果、退出互动（大屏回到签到页面），结束活动（显示活动参与情况的统计结果），退出活动（大屏显示谢谢参与的结束页面）。


**### 访问地址**

服务启动后（默认端口 8000），可以通过以下地址访问：


\*  ***\*大屏幕入口 (Display)\****: [http://ip:8000/](http://ip:8000/)

\*  ***\*管理员入口 (Admin)\****: [http://ip:8000/admin](http://ip:8000/admin)

\*  ***\*手机签到 (Mobile)\****: [http://ip:8000/signin](http://ip:8000/signin)



**## 功能特性**


**### 1. 签到系统**

\*  ***\*扫码签到\****：首页显示动态二维码，用户手机扫号码签到。

\*  ***\*实时统计\****：大屏幕实时显示已签到人数。

\*  ***\*身份识别\****：支持普通参会人和主持人身份（主持人需密码认证）。

\*  ***\*防重复\****：自动识别已签到用户，如果用户已经签到，则再次签到后，不会重复统计签到人数。如果已经开始投票互动，则再次签到后，直接进入投票页面。


**### 2. 后台管理**

\*  ***\*培训配置\****：配置活动主题和投票内容（支持动态添加多次投票及投票内容），需密码访问。

\*  ***\*互动控制\****：主持人并实时控制：开始活动（大屏显示签到页面）、选择开始某一个互动（投票）、结束并显示结果、退出互动（大屏回到签到页面），结束活动（显示活动参与情况的统计结果），退出活动（大屏显示谢谢参与的结束页面）。退出活动后，表示当前活动结束，活动的签到统计信息回自动重置，但当次活动的投票记录等统计信息回自动保存到本地文件中。



**## 技术栈**

\*  ***\*后端\\****: Python 3.10+ + FastAPI + SQLite
\*  ***\*前端\\****: 原生 HTML/CSS/JavaScript
\*  ***\*实时通信\\****: WebSocket
\*  ***\*部署\\****: Docker + Docker Compose

**## 快速开始**

详细安装和使用说明请查看 [QUICKSTART.md](QUICKSTART.md)

### 本地开发

```powershell
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动服务
python backend/main.py

# 3. 访问系统
# 大屏: http://localhost:8000/
# 管理员: http://localhost:8000/admin
# 签到: http://localhost:8000/signin
```

### Docker 部署

```powershell
docker-compose up -d
```
